# Google Play Store - Screenshot Guidelines

## Technical Requirements

### Screenshot Specifications:
- **Format:** PNG or JPEG
- **Minimum dimensions:** 320 × 569 pixels
- **Maximum dimensions:** 3840 × 2160 pixels
- **File size:** Under 8 MB per image
- **Recommended aspect ratio:** 9:16 (vertical/portrait)

### Quantity:
- **Minimum:** 2 screenshots
- **Maximum:** 8 screenshots
- **Recommended:** 5-6 screenshots

## Screenshot Order Strategy

**Best Practice:** Tell a visual story about your app

1. **First Screenshot** - Most important!
   - Shows main feature/interface
   - Captures attention
   - Users see this before clicking into your app

2. **Second Screenshot** - Key differentiator
   - Shows unique feature
   - Why user should choose YOUR app

3-5. **Additional Screenshots** - Feature highlights
   - Different use cases
   - Key functionality
   - User benefits

## Design Tips

✅ **DO:**
- Use real app screenshots (not mock-ups)
- Add text overlays to highlight features
- Use consistent branding colors (#F97316 for Masidy)
- Include captions (100-200 characters each)
- Show actual content/examples
- Use high contrast for readability
- Test on small phones (not just tablets)

❌ **DON'T:**
- Use blurry or low-resolution images
- Include sensitive user data
- Use watermarks or logos larger than necessary
- Make text too small to read
- Mix portrait and landscape
- Use outdated screenshots
- Include personal information

## How to Create Screenshots

### Option 1: From Your Phone
1. Open the Masidy app
2. Navigate through key features
3. Take screenshots (Volume Down + Power button)
4. Edit in your phone's gallery app
5. Add text/captions

### Option 2: From Browser
1. Open your app in a mobile browser simulator
2. Use Chrome DevTools (F12)
3. Click "Device Toggle" (Ctrl+Shift+M)
4. Select "iPhone 12" (390 × 844)
5. Take screenshot (right-click → Screenshot)
6. Edit in an image editor

### Option 3: Design Tool
1. Use Figma or Photoshop
2. Create 390 × 844px artboard
3. Add app UI screenshots
4. Add text overlays
5. Export as PNG

## Text Overlay Best Practices

**Font Recommendations:**
- Sans-serif (clean, modern)
- Minimum 24px size
- High contrast (white text on dark, or dark on light)
- Bold for emphasis
- Max 100-200 characters per caption

**Placement:**
- Bottom 1/4 of screen (where text won't cover key features)
- Center-aligned or left-aligned
- Keep safe margin from edges (20-30px)

## A/B Testing Tips

Different regions may prefer different highlights:
- US/EU: Focus on features & productivity
- India: Focus on value & free aspects
- Asia: Focus on speed & efficiency

You can use different screenshots for different regions in Play Console.

## Accessibility Considerations

- Ensure text has sufficient color contrast (WCAG AA)
- Don't rely solely on color to convey information
- Include text descriptions of visual elements
- Test on different screen sizes

## File Organization

Create a folder structure:
```
screenshots/
├── 1_main_chat.png
├── 2_web_search.png
├── 3_code_generation.png
├── 4_data_analysis.png
├── 5_mobile_responsive.png
└── README.md (notes about each)
```

## Updating Screenshots

- Update when major features change
- Quarterly reviews recommended
- Monitor user feedback about clarity
- A/B test different versions
- Keep version history locally

## Common Mistakes to Avoid

1. ❌ Showing loading screens
2. ❌ Using placeholder text
3. ❌ Including personal data/emails
4. ❌ Low contrast text
5. ❌ Too many different layouts
6. ❌ Outdated UI/features
7. ❌ Text that doesn't match actual app
8. ❌ Mixing different app themes/versions
